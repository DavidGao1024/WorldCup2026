## 方法论

### 核心公式：期望值 (Expected Value)

```
EV = 模型概率 × 赔率 - 1
```

- **EV > 0**：正期望值（价值注），长期买能赚钱
- **EV < 0**：负期望值，体彩占优
- **EV 越高**：投注价值越大

### 分析流程

1. **获取模型概率**：从泊松预测文件中读取每场比赛的胜/平/负概率和比分概率
2. **获取实时赔率**：从 `data/lottery-odds.json` 读取体彩各玩法赔率
3. **计算各选项 EV**：对每个投注选项（胜平负、让球、总进球、半全场、比分），用模型概率 × 赔率计算 EV
4. **排名筛选**：按 EV 排序，找出正 EV 的价值注
5. **串关计算**：两场各选一个选项组合成 2串1，联合概率 = P(A) × P(B)，联合赔率 = odds_A × odds_B

### 概率推导

- **胜平负 (HAD)**：直接使用模型预测的胜/平/负概率
- **让球 (HHAD)**：从比分概率聚合（净胜2+=主胜，净胜1=平，平或负=客胜）
- **总进球 (TTG)**：从比分概率按总进球数聚合
- **半全场 (HAFU)**：基于全场胜/平/负概率 × 上下半场分布系数估算
  - 胜胜≈全场胜×0.55，平胜≈全场胜×0.25，负胜≈全场胜×0.05
  - 平平≈全场平×0.65，胜平≈全场平×0.20，负平≈全场平×0.15
  - 负负≈全场负×0.55，胜负≈全场负×0.25，平负≈全场负×0.15
- **比分 (CRS)**：直接使用模型预测的各比分概率

### 串关说明

- 2串1：两场比赛各选一个选项，两个都中才算赢
- 联合概率 = P(选项A) × P(选项B)（假设两场独立）
- 联合赔率 = 赔率A × 赔率B
- 联合EV = 联合概率 × 联合赔率 - 1
- 奖金 = 2元 × 联合赔率

### 核心发现

**两场平局都被严重低估**：

| 比赛 | 模型平局概率 | 隐含赔率 | 实际赔率 | 价值差距 |

| 法国vs西班牙 | 45% | 2.22 | **3.13** | +40.9% |
| 英格兰vs阿根廷 | 55% | 1.82 | **2.75** | +51.3% |

体彩开出的平局赔率远高于模型隐含概率，两场都有显著正 EV。原因分析：
- 半决赛强强对话，双方保守节奏，90分钟平局概率天然偏高
- 模型考虑了伤病、战术风格等因素，给出比市场更高的平局概率
- 体彩可能过度受"必须分出胜负"的思维影响（实际上90分钟平局很常见）

## 如何重新分析

赔率变化后，只需两步：

### 第1步：更新赔率数据

```bash
node scripts/fetch-odds.js
```

### 第2步：运行分析脚本

```bash
node scripts/analyze-bets.js
```

脚本会读取最新的 `data/lottery-odds.json` 赔率 + `predictions/` 中最新的模型预测，重新计算所有 EV 和推荐方案。

如果模型预测也更新了（比如赛前阵容公布后），需要先更新 `scripts/analyze-bets.js` 中的 `model` 对象里的概率数据，再运行分析。

### 快速检查关键指标

```bash
# 只看正EV选项
node scripts/analyze-bets.js | grep "VALUE"

# 只看2串1推荐
node scripts/analyze-bets.js | grep -A 20 "2串1"
```

## 风险提示

- 模型概率是估算值，存在误差（历史准确率：方向74%，比分19%）
- 正EV不代表单注一定赢，而是长期（大量重复）能盈利
- 半决赛变数大：红牌、点球、伤病突发等都可能改变结果
- **理性投注，量力而行**
