# World Cup 2026 — 赛程比分追踪页面 设计文档

## 概述

纯前端静态页面，托管于 GitHub Pages。展示 2026 年美加墨世界杯赛程、积分榜和淘汰赛对阵信息。数据来源于 OpenFootball 开源数据集，通过 jsDelivr CDN 加载。

## 技术选型

- **纯 HTML/CSS/JS**，零框架、零依赖、零构建
- 数据源：`https://cdn.jsdelivr.net/gh/openfootball/worldcup.json@master/2026/worldcup.json`
- 托管：GitHub Pages
- 无后端，无 API Key

## 页面结构

```
┌─────────────────────────────┐
│  Header: 标题 + 时区选择器    │
├─────────────────────────────┤
│  [ 赛程 ] [ 积分榜 ] [ 淘汰赛 ] │
├─────────────────────────────┤
│          内容区               │
├─────────────────────────────┤
│  Footer: 数据来源             │
└─────────────────────────────┘
```

- 顶部固定 Tab 切换，当前 Tab 高亮
- 时区选择器位于 Header 右侧
- 内容区根据选中 Tab 动态渲染

## 视觉风格

**运动绿茵风**：深绿色主调 `#1a472a` + 金色点缀 `#ffd700`，模拟足球场草地氛围。

## 功能模块

### Tab 1 — 赛程

- 按日期分组展示比赛列表（如 "6月11日 周四"）
- 今日日期自动高亮
- 筛选栏：下拉选择小组或球队过滤
- 每张比赛卡片显示：时间（已转换时区）、队名+国旗 emoji、比分/赛前"vs"、场馆
- 淘汰赛比赛用标签区分

### Tab 2 — 积分榜

- 下拉切换 12 个小组（Group A-L）
- 表格列：球队 | 赛 | 胜 | 平 | 负 | 进球 | 失球 | 净胜 | 积分
- 排序：积分 → 净胜球 → 进球数
- 前两名晋级高亮标记
- 积分从比赛结果实时计算

### Tab 3 — 淘汰赛

- 对阵树：Round of 32 → Round of 16 → 四分之一决赛 → 半决赛 → 三四名 → 决赛
- 桌面端树形展示，移动端横向滚动
- 胜者连线到下一轮
- 未确定球队显示占位文字

### 时区选择器

- 选项：北京时间 (UTC+8) | 美东时间 (UTC-4) | 格林威治 (UTC+0)
- 首次访问根据浏览器时区自动预选
- 选择存入 LocalStorage 持久化
- 所有比赛时间即时转换显示

### 中英双语

- 顶部语言切换按钮
- 界面文字支持中/英两套，球队名和球员名保留英文
- 用户选择存入 LocalStorage

### 球队筛选

- 赛程 Tab 内支持按球队名称筛选
- 输入框搜索 + 下拉选择

## 文件结构

```
/
├── index.html          # 主页面，包含 Tab 和 Header 结构
├── css/
│   └── style.css       # 全局样式，运动绿茵风主题，响应式
├── js/
│   ├── app.js          # 入口：初始化、Tab 切换、全局状态
│   ├── data.js         # CDN 数据加载、解析、缓存 (LocalStorage)
│   ├── schedule.js     # 赛程 Tab：日期分组、筛选、渲染比赛卡片
│   ├── standings.js    # 积分榜 Tab：计算积分、渲染小组表格
│   ├── knockout.js     # 淘汰赛 Tab：构建对阵树、渲染 bracket
│   ├── timezone.js     # 时区转换工具
│   └── i18n.js         # 中英文翻译映射
└── data/
    └── worldcup.json   # 本地备用数据（CDN 不可用时）
```

## 数据流

1. `app.js` 启动 → 检查 LocalStorage 缓存 → 有缓存直接用，无缓存 fetch CDN
2. `data.js` 拉取 JSON → 按 round/group/date 建立索引
3. `standings.js` 从 match 结果计算 12 个小组积分榜
4. `knockout.js` 解析淘汰赛轮次，构建对阵树，引用 match num 连接晋级路径
5. 各 Tab 渲染模块消费处理后的数据
6. 时区变更时通知各模块重新渲染时间字段

## 数据格式 (worldcup.json)

```json
{
  "name": "World Cup 2026",
  "matches": [
    {
      "round": "Matchday 1",
      "num": 1,
      "date": "2026-06-11",
      "time": "13:00 UTC-6",
      "team1": "Mexico",
      "team2": "South Africa",
      "group": "Group A",
      "ground": "Mexico City"
    }
  ]
}
```

- 共 104 场比赛（72 场小组赛 + 32 场淘汰赛）
- 12 个小组（A-L），每组 4 队
- 比赛时间带 UTC 偏移量，便于时区转换
- 淘汰赛队伍引用格式：`W{num}` = 第 num 场比赛胜者，`L{num}` = 负者
- 当前数据为赛前赛程，不含比分字段。比赛期间社区会更新 `score1`/`score2` 字段，代码需同时兼容有无比分的情况
- 少部分淘汰赛前场次缺少 `num` 字段，需按数组顺序补全编号

## 响应式策略

- 桌面（≥768px）：Tab 横向排列，赛程双列卡片，积分榜全宽表格
- 移动（<768px）：Tab 等宽分布，赛程单列全宽卡片，积分榜可横向滚动，淘汰赛横向滑动

## 边界与限制

- 开源数据不包含实时比分更新（需手动合并最新的 JSON 或后续对接 API）
- 国旗使用 emoji 映射表（国旗码 → emoji），需要维护一份国家名到国旗码的映射
- 无后端 = 无用户系统、无推送通知
- GitHub Pages 有 1GB 存储和 100GB/月带宽限制，对静态页面完全足够
